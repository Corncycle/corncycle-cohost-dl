Explanation

So what in the world is going on here? Cohost restricts posts to inline styles, meaning we don't have access to pseudo-classes like nth-of-type for easy modular arithmetic. Also, inline styles means we can't use ::before and ::after pseudo-elements to programmatically print Fizz/Buzz, so we'll have to settle for styling elements in a way that follows the FizzBuzz rules. calc seems tempting, but unfortunately, mod, which is a part of the newest CSS Values Level 4 Working Draft is not yet implemented by most browsers.

I was shopping around for ways to get the math to work. You can implement mod if you have integer division: mod(x, d) = x - d * (x // d) if // is integer (flooring) division. Again, there's a floor function in that CSS working draft, but it isn't supported anywhere, so we can't use it. I found this StackOverflow post by hdante which details an amazing double precision floating point hack. I'll let hdante explain it:

div {
    --shf: 4.9406564584124654e-322;
    width: calc(((100% / 11) - 9.09px) * var(--shf) / var(--shf));
}

What this does: it multiplies the value to be rounded by a really small value that underflows the value starting at the third decimal point. Then, it divides the truncated value back, resulting in a rounded version of the value. This assumes that all browsers you support use 64-bit floating point values. If they don't, not only this will be wrong, it might return zero when using smaller floating point data types, completely breaking your page.

Change the exponent to -323 to round at the first decimal point and -324 to round at integer values.

I've tested this in Chrome and Firefox and it only works in Chrome. Presumably, I need some different magic value for Firefox. Oh well, that's a job for another day.

Not-so-crimes

Now that we have a flooring function, we can implement mod. Above, I'm using two absolutely positioned color divs inside of a fixed-size wrapper div for each number. One div is for Fizz (red), the other is for Buzz (blue). Both are translucent so that in the FizzBuzz case, they can overlap and color the square purple. The color divs have left offsets given by the complicated calc function that makes them only show up when the mod is 0. Here's the relevant code for the Fizz div:

      left: calc(
        (
            {n} -
              (
                3 *
                  (
                    (({n} / 3) * 4.9406564584124654e-324) /
                      4.9406564584124654e-324
                  )
              )
          ) * 2rem
      );

The only caveat is that this calc code has to be repeated for each number, but I consider this method to be a legitimate version of FizzBuzz because the Fizz/Buzz value for each number is resolved via computation by the browser. I wrote a script to generate it, but I don't consider the looping an interesting problem (it's probably not solvable without JavaScript). That's it for my CSS crimes. I'm very proud/ashamed!
